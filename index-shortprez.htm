Types of Web animations

Animated gifs

Frame to frame, indexed color, 8-bit, 256 color or less image

Good

  • Well supported
  • Intervals and looping is controllable

Bad

  • Limited colors, no real transparency
  • Terrible for photography
  • Large file size
  • Unscriptable, no interaction
  • Can't control timing or state

CSS3 transitions

An element change change from one state to another.

.square { border-radius: 0; background-color: rgba(119, 119, 170, 1); transition: all .5s linear; } .square:hover { border-radius: 50%; background-color: rgba(255, 19, 170, 1); }

Good

  • Easy
  • Light
  • Controllable with delays
  • Semi - Scriptable (by adding a class dynamically)
  • IE 10+

Bad

  • Only state to state animations
  • Needs a trigger or event

CSS3 keyframes

An element can change along a timeline.

.keyframes .square { border-radius: 0; animation: roundoff 3s infinite linear; } @keyframes roundoff { 0%, 100% { border-radius: 0; } 50% { border-radius: 50%; transform: translateX(200px); } }

Good

  • Light
  • Timeline based
  • Play state, delays, directions etc.
  • Semi Scriptable (by adding a class dynamically)
  • IE 10+

Bad

  • Confusing timeline code
  • Difficult to sync and combine multiple transforms

CSS3 keyframes

Using the steps() method, we can move
a sprite's background-position so it
works like an animated gif.

.flourbag1 img{ animation: walk .8s steps(8) infinite; } @keyframes walk { 0% { transform: translateX(0px);} 100% { transform: translateX(-764px); } }

Good

  • Animatable jpgs!
  • Play state, delays, directions etc.
  • Semi Scriptable (by adding a class dynamically)

Bad

  • Confusing timeline code

More important UI Principles

  1. Don't make 'em wait'
  2. Don't induce seizures
  3. Don't animate for the sake of animation, make sure it has a purpose
  4. Watch where you're making their eyes go
  5. Things the user needs to get to again should come and leave the same way
  6. Things should enter faster than they leave, for percieved speed
  7. As a beginner make your transitions half as fast as you think they should be (and double your time estimate for doing this stuff).

This is a simple demo to show how to set the transform origin of some curved, rotating text.

The transform-origin of the curved text is set halfway across on the Y axis and 240% down on the X (roughly the center of the circle.)

The next step is to make this scalable. It's probably easier to absolutely nest the text box in the circle, so calculating the center point just takes some radius math.

I've used an arbitary asset for this, but would used a more precise path and image width if it needed to be perfect.
